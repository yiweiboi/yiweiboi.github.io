思路分析

  • 这道题不仅考察对链表的操作，还需要一定的数学运算

  • 我们首先要判断这条链表是否有环：

    • 我们设置快慢指针，如果有环，那么fast指针一定是先进入环的，通过判断fast指针是否为空或者fast.next是否为空来判断是否有环

  • 如果确实有环，我们需要判断环的入口在哪：

    • 首先第一点，如果有环的话，那么快慢指针一定是在环中相遇的，这是毋庸置疑的
    • 其次，我们需要判断环的入口在哪里了
    • 假设头节点到环的入口需要x个节点，环的入口距离快慢指针相遇位置有y个节点，从相遇节点到环的入口有z个节点
    • 相遇时，slow指针已经走了x + y个节点，fast指针已经走了x + y + n（y + z）个节点，n表示fast指针在环内走的圈数
    • 因为fast指针走过的节点树是slow指针的两倍，可以得到**x + y + n（y + z）= 2（x + y） **
    • 化简得到x = （n - 1）（y + z） + z，这里的n一定是大于1的，因为fast至少需要走一圈才能追上slow指针
    • 这个公式说明，头节点到环入口需要的节点个数是n - 1圈加上从相遇节点到环入口的距离
    • 那么我们重新设置一个从头节点开始和相遇节点每次同时移动一个节点，那么它们最终会相遇在环入口节点位置

  • 可能有同学要问：为什么第一次在环中相遇，slow指针走了x + y个节点，而不是x + 若干环长度 + y呢？

    • 我们要知道，在slow指针第一次进入环时，fast指针最多和slow指针相差n - 1个节点，n为环中节点数
    • 我们假设slow指针走了一个环的距离，即n个节点，那么fast指针走了2n个节点
    • 然而，2n > n - 1 + n，说明在slow完整走一个环之前，fast就已经追上slow指针了
